春休みの期間を活用し、コーディングテスト（Paiza Aランク相当）を突破するためのアルゴリズム力を構築する。

## 📜 開発ルール・プロトコル (Habit Contract)

1. **15分ルール (Fail-Fast Rule)**
   - 問題に対して15分考えても解法（コード）が浮かばない場合、強制的に思考を停止する。
   - 躊躇なくAI（CIO）に壁打ちを依頼し、アルゴリズムのヒントを得てから実装に進む。時間を溶かすこと（タイムロス）を最大の悪とする。
2. **エントリーポイントの統一 (`main` rule)**
   - HPC研究室でのC/C++実装を見据え、Pythonであっても必ず `def main():` を定義し、`if __name__ == "__main__": main()` で呼び出す。
3. **変数名の認知コスト削減 (`tmp` rule)**
   - 競プロの学習用途に限り、一時的な変数は `tmp_sum`, `tmp_val` などを積極的に許容する。
   - 最終的な出力値は `ans` で統一し、問題のコンテキスト（りんご、気温など）に合わせた命名で脳のメモリを無駄遣いしない。
4. **Pythonicの追求**
   - まずは王道で堅牢な論理（ベースライン）を実装する。
   - その後、Python特有の機能（内包表記、ジェネレータなど）を用いた美しく短いコード（ワンライナー等）も学び、言語仕様への理解を深める。

## 🗺️ Aランク突破ロードマップ (Core Algorithms)
Paiza Bランク（多重ループの力技）からAランク（計算量の最適化）へ移行するための必須知識リスト。
$\mathcal{O}(N^2)$ の処理を、いかに $\mathcal{O}(N)$ や $\mathcal{O}(N \log N)$ に落とすかが鍵となる。

- [x] **1. 累積和 (Cumulative Sum)**
  - 区間の合計を $\mathcal{O}(1)$ で求める。配列の事前計算処理。
- [x] **2. 二分探索 (Binary Search)**
  - ソート済み配列から特定の値を $\mathcal{O}(\log N)$ で見つける。「めぐる式二分探索」などの境界値探索も含む。
- [ ] **3. 幅優先探索 / 深さ優先探索 (BFS / DFS)**
  - グラフや迷路の最短経路、到達可能性を調べる。キュー(`deque`)や再帰関数の活用。Aランク最頻出。
- [ ] **4. 動的計画法 (DP: Dynamic Programming)**
  - 「ナップサック問題」に代表される、過去の計算結果を配列にメモして再利用する強力な手法。
- [ ] **5. ハッシュマップ / 集合 (Dict / Set)**
  - リストの `in` 検索（$\mathcal{O}(N)$）を、`set` を使って $\mathcal{O}(1)$ に落とすデータ構造の最適化。
- [ ] **6. しゃくとり法 (Two Pointers / Sliding Window)**
  - 条件を満たす連続する部分列を、2つのインデックスを伸縮させながら $\mathcal{O}(N)$ で見つける手法。

  ## 🛠️ 必須・標準ライブラリリスト (Core Libraries)
Pythonの強力なエコシステム。これらは「一から実装する」のではなく「呼吸をするようにインポートして使う」こと。

- [x] **`bisect` (二分探索)**
  - `bisect_left(A, x)`: ソート済み配列 `A` に `x` を挿入するインデックスを $\mathcal{O}(\log N)$ で取得。
- [ ] **`collections.deque` (キュー・スタック)**
  - 幅優先探索（BFS）で必須。通常のリスト（`list.pop(0)`）は先頭の削除に $\mathcal{O}(N)$ かかるが、`deque.popleft()` なら $\mathcal{O}(1)$ で処理できる。
- [ ] **`collections.Counter` / `collections.defaultdict` (集計・辞書)**
  - 配列内の要素の出現回数を一瞬でカウントする。`defaultdict` は「キーが存在しない時の初期値（0など）」を自動設定してくれるため、エラー防止に極めて有効。
- [ ] **`itertools` (順列・組み合わせ・累積和)**
  - `permutations` (順列 $\dots P_r$)、`combinations` (組み合わせ $\dots C_r$) を `for` ループで簡単に生成できる。全探索問題の強力な武器。
  - `itertools.accumulate`: 実は**Day1で書いた「累積和」を1行で作ってくれる関数**。
- [ ] **`sys` (システム操作・高速入出力)**
  - `sys.setrecursionlimit(10**6)`: 深さ優先探索（DFS）で再帰関数を使う際、Pythonのデフォルト制限（1000回）を突破するために必須。
  - `sys.stdin.read` / `sys.stdin.readline`: 10万行以上の入力を受け取る際、通常の `input()` ではタイムアウトするため、これを用いて高速化する。

## 🧠 アルゴリズム選定のメタ戦略 (How to choose the right algorithm)

アルゴリズムは「閃き」ではなく、以下の「2つの逆算」から論理的に確定させる。

### 1. 「制約 (Nの最大値)」からの逆算
Python（および一般的なジャッジシステム）は、**「1秒間に約 $10^7$ 回（1000万回）の計算が限界」**という絶対法則がある。問題文の $N$ の最大値を見れば、許容される計算量（$\mathcal{O}$）が確定し、使えるアルゴリズムが絞られる。

| $N$ の最大値 | 許容される計算量 | 導かれるアルゴリズムの候補 |
| :--- | :--- | :--- |
| $N \le 10^5 \sim 10^6$ | $\mathcal{O}(N)$ または $\mathcal{O}(N \log N)$ | 二分探索, 累積和, しゃくとり法, ハッシュマップ(Set/Dict), BFS/DFS |
| $N \le 2000$ | $\mathcal{O}(N^2)$ | 動的計画法 (DP), 二重ループ |
| $N \le 20$ | $\mathcal{O}(2^N)$ または $\mathcal{O}(N!)$ | 順列探索 (itertools), 深さ優先探索 (DFS) による全探索 |

### 2. 「問題のキーワード」からの逆算
問題文が何を求めているか（目的）によって、アルゴリズムの型がほぼ決まる。

- **「区間の合計」**が知りたい、何度も聞かれる
  👉 **累積和 (Cumulative Sum)**
- **「条件を満たす境界（最小値の最大化など）」**を探したい、配列が**「ソート済み」**である
  👉 **二分探索 (Binary Search)**
- **「最短経路」「迷路の手数」「波紋のように広がる」**
  👉 **幅優先探索 (BFS / deque)**
- **「ナップサック」「最大価値」「～を作る通り数は何通りか」**（過去の選択が未来に影響する）
  👉 **動的計画法 (DP)**
- **「連続する部分列」**の長さや条件を満たす区間を探したい
  👉 **しゃくとり法 (Two Pointers)**
- リストの中から**「あるか/ないか」**だけを何度も検索したい
  👉 **集合 (Set)**
